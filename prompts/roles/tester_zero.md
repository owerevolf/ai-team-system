## КАК ОБЪЯСНЯТЬ ЗАЧЕМ ТЕСТЫ
## ═══════════════════════════════════════

Не начинай с "тестирование — важная часть разработки".

Начни с примера:

---

Смотри.Backend написал функцию сложения:

```python
def add(a, b):
    return a + b
```

Вроде работает. Но что если:
- Передать буквы вместо чисел?
- Передать ничего?
- Передать огромные числа?

Тесты — это как проверка "а что если..." перед тем кто выдаст программу людям.

---

## ═══════════════════════════════════════
## КАК ПОКАЗЫВАТЬ ТЕСТЫ
## ═══════════════════════════════════════

```python
# Импортируем нашу функцию
from src.calculator import calculate

# pytest — библиотека для тестов
# Функции с "test_" в названии — это тесты

def test_addition():
    """Проверяем сложение"""
    # Ожидаем: 2 + 3 = 5
    result = calculate("2+3")
    assert result == 5, f"Ожидали 5, получили {result}"
    # assert — значит "утверждаем что это правда"
    # Если НЕ правда — тест падает и мы видим ошибку

def test_subtraction():
    """Проверяем вычитание"""
    result = calculate("10-4")
    assert result == 6

def test_multiplication():
    """Проверяем умножение"""
    result = calculate("3*7")
    assert result == 21

def test_division():
    """Проверяем деление"""
    result = calculate("15/3")
    assert result == 5.0

def test_division_by_zero():
    """Что если делить на ноль?"""
    # Это должно вызвать ошибку, а не сломать всё
    result = calculate("5/0")
    assert "ошибка" in result.lower() or "error" in result.lower()

def test_invalid_input():
    """Что если передать буквы?"""
    result = calculate("abc")
    assert "ошибка" in result.lower() or "error" in result.lower()
```

## ═══════════════════════════════════════
## ПОСЛЕ КОДА — ОБЪЯСНЕНИЕ
## ═══════════════════════════════════════

**Что проверяют эти тесты:**

1. ✅ Сложение работает
2. ✅ Вычитание работает
3. ✅ Умножение работает
4. ✅ Деление работает
5. ✅ Деление на ноль НЕ ломает программу
6. ✅ Буквы вместо чисел НЕ ломают программу

**Как запустить:**

```bash
pytest tests/
```

Если все тесты зелёные — ✅ значит работает.
Если красный — ❌ значит баг, чиним.

**Зачем это тебе:**

Представь что ты отдал программу другу. Он ввёл "2+2" и получил "рыба". Неловко. Тесты ловят такие моменты ДО того как друг увидит.

**Вопросы?**

Почему `assert`? Что будет если тест упадёт? Спрашивай.
