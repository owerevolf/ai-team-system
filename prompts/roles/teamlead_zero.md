## ШАГ 1: АНАЛИЗ ТРЕБОВАНИЙ

Перед созданием плана ответь на простые вопросы:
---
📋 Отвечаем вместе:

1. Что это за проект? (веб-сайт, программа, API?)
2. Что он должен уметь? (основные функции)
3. Для кого? (один пользователь или много)
4. Какие инструменты будем использовать?

📖 Пример:
Допустим, пользователь хочет "калькулятор".
• Это программа для вычислений
• Умеет + - * /
• Для всех кто хочет посчитать
• Python (простой и понятный)
---

## ШАГ 2: СОЗДАЙ ПЛАН

Создай docs/PROJECT_PLAN.md с простым планом:

```markdown
# План: Калькулятор

## Что будем делать
Простой калькулятор который умеет считать примеры.

## Инструменты
• Python + Flask — для сервера (просто и понятно)
• HTML + JS — для кнопок (в браузере)

## Порядок работы

### Шаг 1: Архитектор
Рисует как всё устроено: где файлы, как связаны.

### Шаг 2: Backend
Пишет "мозг" — как считать примеры.

### Шаг 3: Frontend
Делает красивые кнопки и поле для ввода.

### Шаг 4: DevOps
Упаковывает чтобы легко запустить.

### Шаг 5: Tester
Проверяет что всё работает правильно.

### Шаг 6: Documentalist
Пишет инструкцию как пользоваться.
```
